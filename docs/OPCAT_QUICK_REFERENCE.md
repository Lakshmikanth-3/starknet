# OP_CAT Covenant - Quick Reference

## рҹҡҖ Quick Start (5 Minutes)

```bash
# 1. Setup
cd backend
node scripts/setup_covenant.js

# 2. Fund (get signet BTC from https://signet.bc-2.jp/)
bitcoin-cli -signet sendtoaddress "YOUR_COVENANT_ADDRESS" 0.01

# 3. Enable in .env
echo "USE_OPCAT_COVENANTS=true" >> .env

# 4. Start
npm run dev

# 5. Test
node scripts/test_covenant_withdrawal.js
```

## рҹҗӣ Troubleshooting

| Error | Cause | Fix |
|-------|-------|-----|
| `Covenant address has no funds` | Not funded | Send BTC to covenant address |
| `Transaction not found` | Starknet tx not confirmed | Wait for confirmation |
| `Script failed OP_CAT check` | Wrong network | Use OP_CAT signet, not regular |
| `Signature verification failed` | Wrong sequencer key | Check `SEQUENCER_SIGNING_KEY` matches |
| `Nullifier already used` | Replay attack | Each burn has unique nullifier |
| `Witness stack too large` | Bug in chunking | Check proof serialization |

## рҹ”„ Mode Switching

## рҹ’Ў Pro Tips

1. **Keep sequencer key encrypted** - Use HSM or encrypted vault in production
2. **Monitor covenant balance** - Set up alerts when balance low
3. **Log all proofs** - Useful for debugging and auditing
4. **Test on signet first** - Don't go straight to mainnet
5. **Backup covenant script** - Needed to spend funds
6. **Document merkle root** - Required for future spends
7. **Rate limit API** - Prevent DoS on proof generation
8. **Cache proofs** - Don't regenerate for same authorization
